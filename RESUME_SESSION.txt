---------------------------------------------------------------
NON-NEGOTIABLE RULES
---------------------------------------------------------------
1. Bronze never fails on data content — only infra/file errors fail the job
2. Every source column cast to STRING in Bronze — no type inference
3. Malformed rows go to bronze_corrupt table — never silently dropped
4. Zero hardcoded paths, formats, or dataset names — everything from YAML
5. Storage is pluggable via URI scheme (abfss://, s3a://, gs://, local)
6. Every Bronze row gets these 8 metadata columns:
   _batch_id, _run_id, _source_file_name, _source_file_hash,
   _ingestion_timestamp, _ingestion_date, _record_seq, _source_row_uuid
7. Bronze tables partitioned by _ingestion_date
8. File watermark prevents reprocessing — keyed on content hash not filename

IMPORTANT NOTES
---------------------------------------------------------------
- NEVER commit real Azure storage keys to Git
- Always set key via:
    os.environ["AZURE_STORAGE_ACCOUNT_KEY"] = "<key>"
    spark.conf.set("fs.azure.account.key.medelliondataarcs.dfs.core.windows.net", "<key>")
- .gitignore has *.ipynb to prevent notebook leaks
- Git push issues: always use git push --force origin main
- Databricks Pull issues: use subprocess git reset --hard origin/main
- To sync files from GitHub to Databricks:
    import requests
    url = "https://raw.githubusercontent.com/kinjal12365/Building_a_reusable_medellion_data_architecture/main/<file_path>"
    response = requests.get(url)
    with open('<databricks_path>', 'w') as f:
        f.write(response.text)

---------------------------------------------------------------
CRASH RECOVERY LOGIC (in loader.py)
---------------------------------------------------------------
Before writing any file:
1. Check watermark → if bronze_loaded, skip entirely
2. Check bronze table for partial rows (same _source_file_hash)
3. If partial rows found → delete them → reload fresh
4. Write clean rows → update watermark
Stats now include: files_recovered count
